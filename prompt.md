## Step 1: Analysis and Planning

Before starting implementation, you must first analyze the provided documents and create a comprehensive plan:

### Analysis Requirements

1. **Document Review**: Thoroughly analyze both the Product Requirements Document and Technical Instructions
2. **Technology Stack Assessment**: Identify all technologies, frameworks, and tools mentioned in the technical instructions
3. **Dependencies Identification**: Map out dependencies between different features and components
4. **Phase Breakdown**: Understand the phase-wise structure and what each phase entails

### Planning Output

Provide a detailed plan that includes:

1. **Project Overview**: High-level summary of the entire project and its objectives
2. **Architecture Overview**: Proposed system architecture and component structure
3. **Technology Stack**: Complete list of technologies, libraries, and tools to be used
4. **Phase-by-Phase Breakdown**:
   - List all phases from the PRD
   - Key deliverables for each phase
   - Dependencies between phases
5. **Phase 1 Detailed Plan**:
   - Specific features/components to be built
   - File and folder structure
   - Step-by-step implementation approach
   - Estimated complexity and potential challenges
6. **Development Order**: Logical sequence of building components within Phase 1
7. **Risk Assessment**: Potential challenges or blockers you foresee

## Step 2: Implementation

After completing the analysis and planning, proceed with the implementation.

### Implementation Guidelines

1. **Current Phase**: Implement only Phase 1 initially
2. **Code Quality**: Write clean, maintainable, and well-documented code
3. **Best Practices**: Follow industry-standard coding practices and patterns
4. **File Structure**: Create appropriate file and folder structure
5. **Dependencies**: Use the specified technologies and libraries from the technical instructions
6. **Testing**: Include basic validation and error handling
7. **Documentation**: Add inline comments for complex logic

### Phase Implementation Process

For the current phase, provide:

1. **Phase Overview**: Brief summary of what will be implemented in this phase
2. **File Structure**: Complete directory structure for this phase
3. **Implementation**: Full code for all files needed in this phase
4. **Configuration**: Any required configuration files (package.json, .env.example, etc.)
5. **Setup Instructions**: Step-by-step instructions to run the code
6. **Testing Steps**: How to verify the implementation works correctly
7. **Phase Completion Checklist**: List of completed features/requirements

## Output Format

Provide the response in two clear sections:

1. **ANALYSIS AND PLANNING** - Your detailed analysis and plan
2. **IMPLEMENTATION** - The actual code and setup for Phase 1
